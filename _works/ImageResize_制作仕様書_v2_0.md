# ImageResize 制作仕様書 v2.0

**DesktopKit A-4 — 画像一括リサイズ**

作成：C#開発プロジェクト
日付：2026年3月30日
根拠：制作指示書 v1.0 Phase 5 + 仕様打合せ（2026-03-30）

---

## 1. 概要

### 1-1. このツールの役割

条件を指定して画像ファイルを一括リサイズする。元ファイルは一切変更せず、リサイズ後の画像を並列フォルダに複製保存する。

### 1-2. 想定される主な用途

- AI生成画像の整理（Stable Diffusion、Midjourney、DALL-E、ChatGPT画像生成など）
- Web掲載用・メール添付用の画像縮小
- 大量の画像を同一条件で一括処理したい場面

### 1-3. 設計方針

- **シンプルさ優先**：リサイズと品質の2パラメータだけで完結する
- **元ファイルの安全**：複製保存。元ファイルには一切触れない
- **拡大はしない**：元画像より大きくなる条件は無視する
- **拡張可能な条件エンジン**：初期は％指定のみ。将来の条件追加に耐える構造にする

## 2. 制作指示書 v1.0 からの変更点

本仕様書は、制作指示書 v1.0 Phase 5 の内容を仕様打合せの結果に基づき更新したものである。

| 項目 | 制作指示書 v1.0 | 本仕様書 v2.0 | 変更理由 |
|------|----------------|--------------|---------|
| 対応形式 | JPEG, PNG, BMP（将来WebP） | JPEG, PNG, WebP | AI生成画像の主用途に合わせた。BMPは現在ほぼ使われないため除外。WebPはChatGPT等の出力形式として需要あり |
| 品質パラメータ | なし（リサイズ倍率のみ） | JPEG/WebP品質（％）を追加 | 一括処理で設定項目が1つ増えても操作負荷にならない。ファイルサイズ削減に実効性あり |
| 個別除外 | なし | チェックボックスによる個別除外 | 対象ファイルの取捨選択を可能にする |
| 出力先 | 元フォルダ内にサブフォルダ作成 | 元フォルダと並列にフォルダ作成 | 二重処理防止。複製の安心感。エクスプローラーで元と縮小版を比較しやすい |
| 出力フォルダ名 | 固定ルール | デフォルト自動生成＋利用者変更可 | 利用者の自由度を確保 |
| 画像処理基盤 | System.Drawing | ImageSharp | WebP対応。System.Drawingは古い設計でクセが多い。ImageSharpで3形式を統一的に扱える |
| 形式フィルタUI | チェックボックス3個（.jpg .png .bmp） | チェックボックス3個（.jpg .png .webp） | 対応形式変更に伴う |

## 3. 対応形式

| 形式 | 拡張子 | 品質パラメータ | 特徴 |
|------|--------|--------------|------|
| JPEG | .jpg, .jpeg | あり（デフォルト95%） | 写真向き。非可逆圧縮。品質を下げるとファイルサイズが小さくなるが画質が劣化する |
| PNG | .png | なし（常に劣化なし） | イラスト・透明背景向き。可逆圧縮。品質パラメータは存在しない |
| WebP | .webp | あり（デフォルト95%） | JPEG+PNGのいいとこ取り。同等画質でJPEGより25〜35%小さい |

## 4. 操作フロー

### Step 1：対象を指定

1. 「フォルダを選択」ボタンで画像フォルダを指定する
2. 形式フィルタ（チェックボックス）で対象形式を絞る
   - ☑ .jpg/.jpeg　☑ .png　☑ .webp（すべて初期チェックON）
3. 対象ファイル一覧がDataGridViewに表示される
   - 全ファイルにチェックボックス付き（初期は全件ON）
   - 個別にチェックを外して除外できる

### Step 2：条件を設定

| 設定項目 | コントロール | 初期値 | 備考 |
|---------|------------|-------|------|
| リサイズ倍率 | NumericUpDown | 50 | 1〜100（％）。全形式に適用 |
| JPEG/WebP品質 | NumericUpDown | 95 | 1〜100（％）。JPEG・WebPにのみ有効。対象にJPEG/WebPがない場合はグレーアウトし「※対象にJPEG/WebPファイルがありません」と注釈表示 |
| 出力フォルダ名 | TextBox | 自動生成 | デフォルト：「[元フォルダ名]_[倍率]pct_[yyyyMMdd]」。利用者が自由に変更可能 |
| 出力先 | TextBox + 参照ボタン | 元フォルダの親フォルダ | 通常はデフォルトのまま。変えたい場合は「参照」ボタンで別の場所を選べる |

### Step 3：プレビュー（一覧＋アラート）

DataGridViewの列構成：

| 列 | 内容 |
|----|------|
| ☑ | チェックボックス（除外用） |
| ファイル名 | ファイル名を表示 |
| 形式 | JPEG / PNG / WebP |
| サイズ(KB) | 元ファイルのサイズ |
| 解像度 | W × H ピクセル |
| 推定後サイズ | リサイズ後の推定ファイルサイズ |

アラート色付き表示：

| 色 | 条件 | 意味 |
|----|------|------|
| 赤背景 | 5MB以上 | 巨大ファイル（閾値はデフォルト5MB） |
| グレー背景 | 0バイト | 空ファイル（処理対象外） |

※ 制作指示書 v1.0 の「黄背景：対応形式外」は、形式フィルタで絞る設計にしたことで不要となった

### Step 4：実行

1. 「実行」ボタンを押す
2. 元フォルダと並列に出力フォルダを作成する
3. チェックONのファイルをリサイズし、出力フォルダに保存する
4. 元ファイルは一切変更しない
5. 処理完了後、出力フォルダをエクスプローラーで開く
6. ステータスバーに処理結果を表示（処理件数、スキップ件数）

出力フォルダの構造：

```
C:\画像\                          ← 親フォルダ
  ├── AI作品\                     ← 元フォルダ（指定した場所）
  │    ├── image_001.png          ← 元ファイル（そのまま残る）
  │    ├── image_002.jpg
  │    └── image_003.webp
  └── AI作品_50pct_20260330\      ← 出力先（並列に自動作成）
       ├── image_001.png          ← リサイズ後（PNG：劣化なし）
       ├── image_002.jpg          ← リサイズ後（JPEG：品質95%）
       └── image_003.webp         ← リサイズ後（WebP：品質95%）
```

## 6. 実装クラス

| クラス | 責務 |
|--------|------|
| ImageResizeForm : BaseForm | メインフォーム。条件入力、一覧表示、実行ボタンの配置と制御 |
| ResizeCondition（abstract / interface） | リサイズ条件を表す基底。拡張可能な設計 |
| PercentCondition : ResizeCondition | ％指定リサイズの実装（初期実装） |
| QualitySettings | JPEG/WebP品質パラメータの管理。形式ごとの適用可否判定を含む |
| ImageProcessor | ImageSharpを使用した画像リサイズ処理。Lanczos3リサンプリング |
| FileAlertChecker | ファイルサイズチェック。アラート判定を返す |
| OutputFolderManager | 出力先フォルダの生成・命名・存在チェック |

## 7. 技術要件

### 7-1. 画像処理ライブラリ

| 項目 | 内容 |
|------|------|
| ライブラリ | SixLabors.ImageSharp（NuGetパッケージ） |
| 採用理由 | .NET 8対応、WebP読み書き対応、クロスプラットフォーム、System.Drawingより安定 |
| リサンプリング | Lanczos3（高品質。HighQualityBicubicに相当） |
| JPEG品質 | JpegEncoder.Quality で指定 |
| WebP品質 | WebpEncoder.Quality で指定 |
| PNG | PngEncoder（品質パラメータなし、デフォルト設定で保存） |

### 7-2. 保存時の形式ルール

- 元の形式のまま保存する（JPEG→JPEG、PNG→PNG、WebP→WebP）
- フォーマット変換は行わない
- JPEG/WebPの品質パラメータは利用者の指定値を使用する
- PNGは品質パラメータを無視する（可逆圧縮のため不要）

### 7-3. 拡大禁止ルール

- リサイズ後のピクセル数が元画像以上になる場合、そのファイルをスキップする
- スキップしたファイルはステータスバーに件数を表示する
- 100%指定は実質コピーとなるが、これもスキップ対象とする（リサイズの意味がないため）

## 8. 拡張の余地（のりしろ）

以下は初期実装に含めないが、設計として余地を残す：

| 拡張候補 | 内容 |
|---------|------|
| WidthCondition | 横幅を統一ピクセルに指定 |
| HeightCondition | 縦幅を統一ピクセルに指定 |
| SizeThresholdCondition | 指定KB以上のファイルだけ対象にする |
| フォーマット変換 | WebP→JPEG等。別ツール化の可能性もあり |
| 複数条件の同時実行 | 条件セットの保存・呼び出し |
| BMP対応 | 需要が出た場合に追加（ImageSharpは標準対応済み） |
| 画像内容分類 | 別アプリとして構想中。ImageResizeには含めない |

## 9. 完了条件

1. フォルダ指定 → 形式フィルタ → 一覧表示 → 条件設定 → リサイズ実行の一連の流れが動作すること
2. JPEG、PNG、WebPの3形式がすべてリサイズできること
3. 元ファイルが変更されないこと（複製保存の確認）
4. JPEG/WebP品質パラメータが保存画像に反映されていること
5. PNG処理時に品質パラメータがグレーアウトされること
6. 個別除外（チェックボックスOFF）が機能すること
7. アラート色付き表示が正しく機能すること
8. 出力フォルダが元フォルダと並列に作成されること
9. 出力フォルダ名が変更可能であること
10. 拡大になる場合にスキップされること
11. 処理完了後にエクスプローラーで出力フォルダが開くこと
12. ビルドエラー0で通ること
